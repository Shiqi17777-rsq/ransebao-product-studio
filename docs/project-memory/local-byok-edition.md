# 个人本地版 BYOK 决策

## 产品定位

- 当前媒体 API 扩展先按个人本地版实现。
- BYOK 表示 Bring Your Own Key，用户自备模型 API Key。
- 第一阶段不建设染色宝后端，不托管模型密钥，不统一承担模型调用额度。

## 实现规则

- API Key 只保存在用户本机 `runtime/config/local.json`。
- 客户端可以直接调用用户自己配置的模型 API。
- GitHub 仓库、打包产物、诊断包和项目记忆库都不能包含真实 API Key。
- 设置页必须明确提示：Key 仅本机保存，不上传服务器。
- 初始化向导也提供 Nano Banana Pro / Gemini Key 配置入口，方便第一次使用时一次配齐。

## 商业和体验边界

- 优点：上线快、服务器成本低、密钥风险由用户自己的模型平台账号承担。
- 缺点：用户需要自己申请 API Key，模型额度、地区访问、权限问题也由用户侧解决。
- 适合当前 Windows 本地化第一版和早期专业用户测试。

## 后续升级触发点

只有当出现以下需求时，再考虑后端：

- 普通用户不愿意自己申请 API Key。
- 需要统一付费、计费、额度、团队账号。
- 需要服务端任务队列、统一日志、跨设备同步。
- 需要隐藏公司自己的模型供应商密钥。
