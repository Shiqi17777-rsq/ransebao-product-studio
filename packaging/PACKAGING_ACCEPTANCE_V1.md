# 染色宝第一版本地化打包验收清单

## 当前阶段目标

当前只收一件事：

- **本地化打包第一版是否可交付测试**

这轮先不评估：

- 自动更新
- 热更新
- 远程配置同步
- 云端账号体系

## 优先级

### P0：必须通过

1. Windows 便携版可启动
2. Windows 安装版至少可交付备用
3. 首启向导可进入
4. `runtime` 自动写入用户目录
5. `sau` 可安装
6. `patchright Chromium` 可准备
7. `Dreamina` 可安装并登录
8. 依赖状态文件一致
9. 可完成一轮：
   - 上游
   - 3 图生成
   - 3 图发布

### P1：建议通过

1. Windows 安装版不再因为卸载器写入报错
2. Windows smoke test / diagnostics 工具稳定可复用
3. macOS `.app` 首启链路稳定

### P2：后续再做

1. `.dmg` 精修
2. 签名
3. 自动更新
4. 工作流热更新

## Windows 本轮测试重点

### 只看这 4 件事

1. 包能不能打开
2. 依赖能不能装完
3. 状态文件是否同步
4. 3 图生成和发布能不能走通

### 不用在这轮纠结的

1. 安装体验是否完美
2. 更新机制
3. UI 再优化
4. 高级封装和商业化扩展

## 验收产物

每轮 Windows 测试至少保留：

1. 安装包或便携包版本号
2. diagnostics zip
3. 依赖日志
4. 依赖状态 JSON
5. 一轮生成 / 发布结果截图

## 通过标准

如果 Windows 上满足下面条件，就认为第一版打包可以进入下一阶段：

1. 便携版稳定
2. 安装版至少可作为次选
3. 首启依赖安装不再反复卡死
4. 状态同步不再乱
5. 真实业务链能跑通一次

一句话：

**先把“别人电脑能装、能开、能生成、能发布”做实，再考虑后续更新机制。**
